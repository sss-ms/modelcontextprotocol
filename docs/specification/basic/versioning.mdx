---
title: "版本控制"
description: "Model Context Protocol 版本控制和兼容性管理"
---

:::info
**协议版本**: 2024-11-05
:::

Model Context Protocol 使用基于字符串的版本标识符，格式为 `YYYY-MM-DD`，用于指示最后一次向后不兼容更改的日期。

当前协议版本是 2024-11-05。[查看所有修订版本](/docs/specification/revisions)。

:::info
当协议更新时，只要更改保持向后兼容性，协议版本就*不会*递增。这允许在保持互操作性的同时进行增量改进。
:::

版本协商发生在[初始化](/docs/specification/basic/lifecycle#initialization)阶段。客户端和服务器可以（**MAY**）同时支持多个协议版本，但它们必须（**MUST**）就会话使用的单个版本达成一致。

如果版本协商失败，协议提供适当的错误处理，允许客户端在无法找到与服务器兼容的版本时优雅地终止连接。
